A dive in functional programming for JS

Oct 17 2016

With the time I have exploring JavaScript and its depths, I haven't really focused on learning a particular style of programming. Towards the beginning of my exploration I just focused on learning what the language has to offer.

This took me down the road of learning objects, and finding out that essentially everything in JavaScript is an object that has a prototype. Each prototype can have a large amount of properties and methods, and it's up to you in when you need them. So I set out and got introduced to this style of programming from the one and only, Youtube!

There is a channel called Funfunfunction and he has a series on functional programming in JS. While going a bit deeper in the topic I wanted to read a book focused on this concept, so I found a great book called Mostly Adequate Guide to Functional Programming by Professor Franklin Risby. Now I had two awesome resources and I blew right through them, and this is my run down on functional programming in JavaScript.

Functional programming lends itself to making you a better programmer, that can write cleaner and more secure code. The advantages that come from learning this style is composing code that has less errors, and less time overall in programming. Less errors because the code that you produce is easier to read, and less time due to code re-use.

One awesome usage in JavaScript is a concept called Higher Order Functions. Since every function in JavaScript has a value, we can set that value to a variable or we can pass into other functions. 

            var subtractTen = function(x) {
                return x - 10
            }
            var age = subtractTen(32);

            console.log(age); //22

In the example I am showing you that a function can be set to the variable age, and displayed with the value of 22. Now this is where we have some fun with the use of higher order functions. 

                    var planets = [
                        {name: "Mercury", body: "rock"},
                        {name: "Venus", body: "rock"},
                        {name: "Earth", body: "rock"},
                        {name: "Mars", body: "rock"},
                        {name: "Jupiter", body: "gas"},
                        {name: "Saturn", body: "gas"},
                        {name: "Uranus", body: "ice"},
                        {name: "Neptune", body: "ice"}
                    ];
                    var gasGiants = [];
                    for (let i = 0; i < planets.length; i++) {
                        if (planets[i].body === 'gas') {
                            gasGiants.push(planets[i]);
                        }
                        
                    }
                    console.log(gasGiants); // {Object},{Object}

This example is without the use of JavaScript's built in functions. Essentially the example has an array of objects with two properties, and they're looped through finding if their body is gas. If that is true, it gets pushed into the gasGiants array. Next we use the power of higher order functions. 

                var planets = [
                    {name: "Mercury", body: "rock"},
                    {name: "Venus", body: "rock"},
                    {name: "Earth", body: "rock"},
                    {name: "Mars", body: "rock"},
                    {name: "Jupiter", body: "gas"},
                    {name: "Saturn", body: "gas"},
                    {name: "Uranus", body: "ice"},
                    {name: "Neptune", body: "ice"}
                ];
                    var gasGiants = planets.filter(function(planet) {
                        return planet.body === 'gas';
                    });
                        
                    console.log(gasGiants); // {Object},{Object}

In this example a filter function is used from the prototype array object, and its parameter is a callback function. Inside the callback function the object that has a matched value is returned. This unlocks the concept of function composition, which is two or more functions combined to produce a new function. One function being the method, and the other being the parameter or callback. Now if you view each example you will see that the second is easier to read, and the logic is shorter.
